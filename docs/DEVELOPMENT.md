# 开发指南

> 本文档记录项目开发过程中的常见问题和最佳实践。

## 常见问题与解决方案

### 1. 修改代码后样式丢失，需要重启 dev 服务器

**问题描述**：
修改 `.tsx` 文件中的 Tailwind CSS 类名后，页面样式丢失（无颜色、无间距、无排版等），必须重启 `pnpm dev` 才能恢复。

**原因**：
- Next.js 开发服务器的热更新（HMR）在某些情况下无法正确更新 Tailwind CSS 编译缓存
- `.next` 目录的构建缓存变得不一致

**解决方案**：

#### 方案 A：完全清理缓存（推荐）
```bash
rm -rf .next node_modules/.cache
pnpm dev
```

#### 方案 B：快速重启
```bash
# 按 Ctrl+C 停止当前进程
# 然后重新启动
pnpm dev
```

#### 方案 C：增量清理
如果只想清理 Next.js 缓存：
```bash
rm -rf .next
# 服务器会自动重新构建
```

**预防措施**：
- 在本地开发时 **始终使用 `pnpm dev`**，不要用 `pnpm build`
- 批量修改后再验证，避免频繁的小改动触发缓存问题
- 如果修改了样式关键文件（如 `app/globals.css`），最好主动重启服务器

## 构建和验证命令

| 命令 | 用途 | 场景 |
|------|------|------|
| `pnpm dev` | 启动开发服务器（支持 HMR） | 日常开发 |
| `pnpm build` | 生产环境构建 | 部署前验证 / CI 检查 |
| `pnpm type-check` | TypeScript 类型检查 | 代码审查 / 提交前 |
| `pnpm lint` | ESLint 代码检查 | 代码审查 / 提交前 |

---

## 提交前清单

1. ✅ 在 `pnpm dev` 中验证改动（不只是 build）
2. ✅ 运行 `pnpm type-check` 检查类型
3. ✅ 清理缓存（如遇到样式问题）
4. ✅ 测试相关页面（详情页、侧边栏等）
5. ✅ 验证响应式布局（移动端 / 平板 / 桌面）
